### Code review verdict (src/)
The stack is good. Specific findings:

**Solid**
- `metrics.py` reimplements the FREUID score correctly (DET-based, pinned AuDET, interpolated
  APCER@1%BPCER) with sanity tests.
- `augment_recapture.py` is the key asset: modular print-and-capture simulator (down-up resample +
  double JPEG + blur macro, moiré/halftone/glare/vignette/paper/color-cast/sensor noise), applied
  to **both classes** so "looks recaptured" can't become a fraud shortcut.
- `cv.py` LODO design and the "never greedy-select ensemble members" rule are the right lessons.
- `evaluate.py` correctly notes the metric is rank-based → per-model calibration is cosmetic;
  rank-averaging for ensembles is the right default.
- `infer_ensemble.py`: TTA on logits, rank/geomean combining, aligns to `sample_submission.csv`
  and fills the 135k missing IDs with 0.5 (correct while they are unscored — see §3.6).

**Weak points to fix**
1. **Model selection signal is broken in `train_fast.py`**: it selects the best epoch on
   recap-FREUID of a 5% *random in-domain* split. In-domain is saturated (≈0.000x), so selection is
   noise. Select instead on the **MIDV-Holo OOD probe** (`midv_holo_val.csv` — real captures, real
   presentation attacks, unseen doc types) plus the recap-augmented split as tie-break.
2. The **20 real recaptured train rows** are currently diluted into training. Exclude them from
   training and score them every epoch as a micro stress set (they are the only *real*
   analog-hole samples we have), or at minimum report them separately.
3. Forensic **DCT** variant would see resized pixels (JPEG block grid destroyed by
   `LongestMaxSize`); the planned `forensic_noise` variant is unaffected. Don't use `forensic_dct`
   on resized inputs.
4. Local AuDET pinning conventions may differ slightly from the organizers' implementation —
   calibrate expectations against the public LB with the first submission.

## 2. Where this competition is won

Everyone will hit ~0 on the 5 public digital types. Ranking spread will come from:

1. **Print-and-capture / recapture** (94.5% of test unreleased; organizers state it's the focus)
2. **GenAI multimodal edits** (semantic inconsistencies, not pixel noise)
3. **2 unseen document types** (Asian/African, Latin+Arabic scripts)
4. **APCER@1%BPCER** — the strict operating point. One heavy tail of bona-fide false alarms wrecks
   the harmonic mean even if AuDET is great. Robust, ensembled, rank-averaged scores are the way
   to keep the bona-fide tail clean.

Everything below is prioritized against these four axes.

### P1 — Synthesize the missing attack surface (2–3 days, big differentiator)
The train set contains zero GenAI edits and 20 recaptures; the test emphasizes both. Nobody's
model will have seen FREUID-style attacks on recaptured substrates unless they make them:
- [ ] **Programmatic tamper synthesis on FREUID bona-fides**: portrait swap between two documents
  of the same type; field-level copy-move (dates, MRZ-like lines); text splice from a donor doc
  with slight color/scale jitter → label 1. Push a random subset through `RecaptureSimulator`
  afterwards so "tamper then recapture" artifacts exist in training. This is cheap (cv2 only,
  the doc layouts within a type are aligned enough) and directly targets axes 1+2 combined.
- [ ] **A physical round-trip micro-set**: print 30–50 train images (mix of labels), photograph
  them with a phone, and use as a second *real* stress probe. One evening of work, converts the
  biggest unknown (does the simulator match reality?) into a measurement.

- [ ] **pHash/embedding near-duplicate scan** of `public_test` vs `train` — any near-dupes give
