## Code changes made

- `src/train_fast.py`
  - Added `--ood_csv` support. When a labeled OOD probe is available, best checkpoint selection is now OOD FREUID first, simulated-recapture FREUID second.
  - Added `--exclude_real_recapture_from_train`; FREUID `is_digital=0` rows can now be held out from training and reported every epoch as a real recapture stress set.
  - Added per-epoch history CSV output: `checkpoints/<tag>_history.csv`.
  - Added deterministic Python/numpy/torch seeding and seeded DataLoader workers.
  - Added `--limit_train_batches` / `--limit_eval_batches` smoke controls to validate plumbing without running a full epoch.
  - Added `--no_pretrained` for smoke tests that must not download weights.

- `src/dataset.py`
  - Recapture augmentation now uses the worker-level numpy RNG instead of `np.random.default_rng()` per sample.
  - Disabled Albumentations online version checks with `NO_ALBUMENTATIONS_UPDATE=1`.

- `run_all.py`
  - Full ensemble training now passes `--ood_csv midv_holo_val.csv` when present.
  - Full ensemble training now excludes the 20 real recaptured FREUID rows from train/val and reports them as a stress set.
  - Added resumability: existing target checkpoints are skipped unless `--force` is set.
  - Added `--only`, `--no_infer`, and `--continue_on_error`.
  - Revised the first-pass training budget after a live throughput check:
    - ConvNeXtV2-B: 3 epochs, `p_recapture=0.35`.
    - DINOv2-B: 3 epochs, `p_recapture=0.35`.
    - forensic-noise nano: 3 epochs, `p_recapture=0.50`.
    - workers increased from 6 to 8; eval batch size set to 16.

## Training throughput adjustment

An initial `run_all.py` launch with the old 5/5/4 epoch budget was stopped before epoch 1 completed. It reached only about 4% of the first ConvNeXt epoch after nearly 3 minutes and showed repeated data/augmentation stalls. At that speed, the old budget would likely exceed the intended GPU window. No full-model checkpoint was written.

The runner was therefore changed to the lean first-pass budget above. The objective is to get all three diverse members trained and OOD-selected first, then spend any remaining GPU time on the member that actually improves MIDV-Holo and real-recapture stress metrics.
